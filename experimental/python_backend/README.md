# Experimental Python Backend

This folder contains a separate Python implementation of the target-code generation phase.

## Purpose

- It is intentionally kept separate from the main C compiler.
- It demonstrates a meaningful compiler phase implemented in another language.
- It consumes TAC generated by the main compiler and emits stack-VM style target code.

## Files

- [python_target_codegen.py](/D:/Uni/Semester%208/CC/compiler_construction/experimental/python_backend/python_target_codegen.py)
- [test_python_target_codegen.py](/D:/Uni/Semester%208/CC/compiler_construction/experimental/python_backend/test_python_target_codegen.py)

## Usage

### From raw TAC

```powershell
Get-Content tac.txt | python experimental/python_backend/python_target_codegen.py
```

### From full compiler output

The script can also extract the TAC section from the complete compiler output automatically:

```powershell
@'
int add(int a, int b) {
    return a + b;
}
int main() {
    return add(5, 10);
}
'@ | .\bin\compiler.exe | python experimental/python_backend/python_target_codegen.py
```

## Notes

- This backend is parallel to the C implementation; it is not wired into the main executable.
- Because TAC does not explicitly encode parameter binding names at function entry, the Python backend assumes function parameters are already available as named values when a function begins.
